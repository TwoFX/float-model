# Test vectors

Test vectors for the float model, all in Berkeley **TestFloat** format and stored
gzip-compressed. `lake exe testfloat-check` (see `FloatModelTests/TestFloatCheck.lean`)
discovers every `*.txt.gz` file here, decompresses it with `gzip -dc`, and checks the
model's `binary32`/`binary64` operations against it. There is no on-the-fly generation:
these files are the source of truth.

Each line is `<operand1> [<operand2>] <expected> <flags>` (the second operand is absent for
unary operations), all fields hexadecimal, floats given as their bit patterns; 16 digits
for `binary64`, 8 for `binary32`. The flags field is ignored (the model computes no
exception flags) and NaN results are compared as a class (the model emits a canonical NaN).

File names are `<precision>_<operation>` (e.g. `f64_add`, `f32_sqrt`); the subdirectory
names the suite. Operations covered: `add sub mul div sqrt eq le lt`.

## Suites

### `testfloat/`: Berkeley TestFloat, level 1

Generated by `testfloat_gen` from Berkeley TestFloat
(version 3e, obtain from https://www.jhauser.us/arithmetic/TestFloat.html):

```
testfloat_gen -level 1 -rnear_even <precision>_<operation>
```

### `ucblib/`: UCBTEST library vectors

The `binary32`/`binary64` vectors shipped with the UCBTEST suite (obtain from https://netlib.org/fp/index.html)
(`ucb/ucblib/{add,sub,mul,div,sqrt}{s,d}.input`), converted from their
native format to TestFloat format: only round-to-nearest (`n`) rows are kept (the model
implements no other rounding mode), the two 32-bit words of each `binary64` value are
combined into one bit pattern, and `uo` ("result is unordered/NaN") rows are emitted with a
canonical NaN as the expected value (10 files).

### `ucbtest-gen/`: UCBTEST-generated hard cases

Number-theoretically hard halfway and nearly-halfway cases for multiplication, division,
and square root, generated by adaptations of `ucbmultest`/`ucbdivtest`/`ucbsqrtest` (Kahan's
constructions) that emit TestFloat-format vectors instead of testing the host arithmetic.
Each emitted vector is cross-checked against correctly-rounded host arithmetic, so it is
correct by construction. Default count `ntests = 10000` (6 files: `f32`/`f64` × `mul`/`div`/`sqrt`).

Note: `f32_mul` contains only a handful of vectors. The nearly-halfway multiplication
construction needs intermediates wider than the 24-bit `binary32` significand, so its
provability check correctly declines almost all of those cases in single precision; only the
small halfway set remains. (`f32_div`/`f32_sqrt` are unaffected.) Broad multiplication
coverage for `binary32` comes from the `testfloat/` and `ucblib/` suites.
